# Story 5.2: Dynamic Contextual Reranking via Multi-Armed Bandits

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a data scientist,
I want to apply a Multi-Armed Bandit (MAB) algorithm to dynamically adjust recommendation contextual boosts based on customer click/acceptance feedback,
so that the system continuously learns the optimal promo and time boosts instead of using hardcoded coefficients.

## Acceptance Criteria

1. **Context Representation**:
   - The bandit must represent context using features:
     - `Promo_Active`: Candidate item belongs to an active promotion (0 or 1).
     - `Time_Active`: Candidate item matches the peak hour category (Lunch/Dinner category match) (0 or 1).
2. **Bandit Algorithm**:
   - Implement a Contextual Multi-Armed Bandit (e.g. LinUCB or Contextual Beta-Binomial Thompson Sampling).
   - Expected boost values for `Promo_Boost` and `Time_Boost` must be computed dynamically from the bandit's current parameters.
   - For example, under Thompson Sampling, maintain success/failure parameters $\alpha, \beta$ for promo and time contexts:
     - $\text{Promo\_Boost\_Sample} \sim \text{Beta}(\alpha_{promo}, \beta_{promo})$ scaled to a reasonable range.
     - $\text{Time\_Boost\_Sample} \sim \text{Beta}(\alpha_{time}, \beta_{time})$ scaled to a reasonable range.
   - Under LinUCB, maintain covariance matrix $A$ and reward vector $b$ to estimate the expected click probability for each context.
3. **Stateless API Feedback Loop**:
   - Expose a new endpoint: `POST /api/recommend/feedback`.
   - Request schema:
     ```json
     {
       "recommended_item": "String",
       "accepted": "Boolean",
       "context": {
         "promo_active": "Boolean",
         "time_active": "Boolean"
       }
     }
     ```
   - Processing feedback must execute a stateless parameter update and save the updated weights.
4. **Weights Persistence**:
   - Persist bandit weights locally to `bandit_weights.json` in `_bmad-output/data/` (or project root).
   - If the file is missing, initialize with priors that yield initial boosts matching the original hardcoded values (`+0.20` for promo, `+0.15` for time) to ensure smooth transition.
5. **Backtest Integration**:
   - Update `backtest.py` to simulate bandit online learning:
     - Loop through `orders.csv` transactions.
     - Get recommendation using MAB-adjusted boosts.
     - Simulate customer response.
     - Call the bandit update function immediately with the outcome (reward = 1.0 if accepted, 0.0 if not).
     - Output the final learned parameters and resulting AOV uplift in the console and API responses.
